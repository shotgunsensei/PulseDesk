Take the existing PulseDesk multi-tenant application and build a polished MICROSOFT 365 / ENTRA ID VALIDATION ENGINE plus a TENANT ONBOARDING CHECKLIST PROGRESS TRACKER for the tenant admin experience.

IMPORTANT:
This is not a redesign.
This is a product-hardening and onboarding-acceleration pass focused on making Microsoft 365 integration easier to validate, easier to understand, and easier to operationalize across many healthcare tenants.

PRIMARY GOAL:
Create a tenant-scoped validation and readiness system that helps a PulseDesk tenant admin determine whether their Microsoft 365 / Entra integration is fully configured and ready for real user sign-in.

SECONDARY GOAL:
Create an onboarding checklist progress tracker that makes implementation status visible and reduces confusion for both client admins and PulseDesk implementers.

MULTI-TENANT SAFETY REQUIREMENTS:
- all validation results must be tenant-scoped
- all onboarding progress data must be tenant-scoped
- no tenant can see another tenant’s readiness state, auth config, test results, mappings, or onboarding notes
- all test actions must execute only against the active tenant’s configuration

MAIN FEATURES TO BUILD:
1. Microsoft 365 Configuration Validation Engine
2. Onboarding Checklist Progress Tracker
3. Readiness Status / Readiness Score Card
4. Copy-Friendly Implementation Summary
5. Guided Troubleshooting States
6. Tenant Admin UX improvements around setup confidence

VALIDATION ENGINE REQUIREMENTS:
Build a tenant-scoped validation engine that can assess whether the Microsoft 365 / Entra integration appears correctly configured.

The validation engine should support checks such as:
- auth mode selected
- tenant ID present
- primary domain present
- client ID present
- client secret configured
- redirect URI present and well-formed
- post-logout redirect URI present if required
- at least one allowed domain configured if that control is enabled
- role mappings present
- all core PulseDesk roles mapped or intentionally left unmapped with warning
- JIT provisioning setting configured
- local fallback admin presence if hybrid mode is enabled
- config save state is valid
- last successful configuration test exists
- last successful sign-in test exists if implemented later

VALIDATION OUTPUT REQUIREMENTS:
For each validation item, provide:
- status: passed / warning / failed / not configured
- short explanation
- recommended next step
- quick link to the relevant section if possible

Do NOT expose raw underlying system errors if avoidable.
Translate validation output into clean tenant-admin-friendly language.

READINESS STATUS / SCORE REQUIREMENTS:

- Production Ready (optional if appropriate)

The readiness experience should feel polished and helpful, not gimmicky.

ONBOARDING CHECKLIST TRACKER REQUIREMENTS:
Create a structured onboarding checklist for tenant admins implementing Microsoft 365 sign-in.

Checklist items should include steps such as:
1. Choose auth mode
2. Enter Entra tenant details
3. Provide application credentials
4. Confirm redirect URI / callback setup
5. Create or verify PulseDesk role groups
6. Map Microsoft groups to PulseDesk roles
7. Configure allowed domains if used
8. Enable JIT provisioning if desired
9. Set up fallback local admin if using hybrid mode
10. Run configuration test
11. Test sign-in with admin user
12. Test sign-in with staff/technician user
13. Test unauthorized user behavior
14. Review troubleshooting or warnings
15. Mark tenant ready for go-live

TRACKER BEHAVIOR REQUIREMENTS:
- automatically mark steps complete where possible based on actual configuration state
- allow manual notes or manual confirmation where appropriate
- visually show complete / incomplete / warning states
- show step descriptions and why each step matters
- show blockers clearly
- support progress persistence for the tenant

CHECKLIST UX REQUIREMENTS:

- progress bar or completion count
- quick links to settings sections
- helpful inline guidance
- no clutter
- desktop and tablet friendly
